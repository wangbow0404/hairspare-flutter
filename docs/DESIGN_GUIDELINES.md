# 앱 디자인 룰 (Design Rules)

> 출처: 독립D 유튜브 "앱 디자인 시 꼭 알아야 할 UX/UI 원칙 TOP 10" 요약 정리 (원본: `/Users/yoram/헤어스페어 앱디자인/design-rules.md`)
> 목적: 헤어스페어 앱 리디자인 작업 시 일관되게 지킬 기준. 이 문서에 어긋나는 화면을 새로 만들지 않는다.

---

## 1. 레퍼런스 기준
- 핀터레스트 이미지가 아니라 **이미 출시되어 많은 사람이 쓰는 앱**을 레퍼런스로 삼는다. (예: 토스, 당근마켓, 네이버페이)
- 벤치마킹할 화면은 스크린샷을 떠서 폰트 크기 / 여백 / 버튼 크기를 실측하고 따라 그려본다.

## 2. 폰트 크기 & 줄간격
| 항목 | 기준 |
|---|---|
| 본문 최소 크기 | 13pt |
| 본문 권장 크기 | 14~16pt |
| 절대 하한선 | 12pt (이 밑으로 내려가지 않기) |
| 본문 줄간격(line-height) | 폰트 크기의 1.4~1.6배 |
| 제목처럼 굵은 텍스트 줄간격 | 폰트 크기의 1.2배 |

- 텍스트가 많다고 글자 크기부터 줄이지 않는다. 가독성이 최우선.

## 3. 정보 위계(Hierarchy) — 3단계
- 모든 텍스트를 같은 크기/굵기로 두지 않는다.
- **제목(크게) → 본문(중간) → 부가정보/날짜/조회수(작게)** 3단계로 구분한다.
- 크기, 굵기, 색상, 여백을 조합해서 시선의 흐름을 만든다.

## 4. 아이콘 & 이미지 통일감
- 여러 사이트에서 아이콘을 긁어 섞어 쓰지 않는다 (두께/모서리 모양이 달라 이질감 발생).
- **하나의 아이콘 세트만 사용** (예: Feather Icons, Google Fonts Icons).
- 사진/일러스트도 톤, 채도, 색감을 통일한다.

## 5. 버튼 — 터치 영역 & 피드백
- 최소 터치 영역: **32px 이상**
- 버튼 주변에 충분한 여백 확보 (오터치 방지)
- 버튼을 눌렀을 때 반드시 반응을 준다: 색상 변화, 로딩 스피너 등
- 반응이 없으면 사용자는 "앱이 멈췄다"고 느낀다.

## 6. 색상 제한
- 메인 컬러: **1~2개**
- 보조 컬러: **3개 이하**
- 텍스트–배경 명암 대비: **WCAG 기준 4.5 이상** (webaim.org 대비 체커로 검증 가능)

## 7. 여백(Whitespace) & 그룹핑
- 여백은 정보를 그룹으로 묶어주는 장치 (게슈탈트 근접성 원칙: 가까운 요소 = 같은 그룹으로 인식)
- 관련 있는 요소는 가깝게, 관련 없는 요소는 멀게 배치
- 간격은 **4px, 8px의 배수**로 통일된 spacing 시스템 사용
- 여백이 좁아 보일까 걱정될 땐 좁히기보다 과감히 넓힌다.

## 8. 한 화면 = 하나의 액션
- 한 화면에 기능을 몰아넣지 않는다.
- 복잡한 플로우는 단계별로 쪼개서 **한 화면당 하나의 액션만** 요구한다. (예: 토스 회원가입 원페이지 방식)

## 9. 플랫폼 고유 패턴 준수
- iOS: 좌상단 뒤로가기 버튼
- Android: 하단 내비게이션 바
- 사용자가 이미 익숙한 플랫폼 규칙을 임의로 바꾸지 않는다 → 혼란·이탈 방지

## 10. 실기기 테스트 생활화
- 디자인 툴/개발 환경 화면과 실제 폰 화면은 다르게 보인다.
- 작업 중간중간 **피그마 미러링 기능** 등으로 실기기 확인 습관화.

## 체크리스트 (작업 전 최종 점검용)
- [ ] 본문 폰트 13pt 이상, 줄간격 1.4~1.6배인가?
- [ ] 텍스트 위계가 3단계로 명확히 구분되는가?
- [ ] 아이콘/이미지가 한 세트/한 톤으로 통일됐는가?
- [ ] 버튼 터치 영역 32px 이상 + 클릭 피드백이 있는가?
- [ ] 메인 컬러 1~2개, 명암비 4.5 이상인가?
- [ ] 여백이 4/8 배수 규칙을 따르는가?
- [ ] 한 화면에 액션이 하나뿐인가?
- [ ] iOS/Android 고유 패턴을 그대로 따랐는가?
- [ ] 실기기에서 확인했는가?
