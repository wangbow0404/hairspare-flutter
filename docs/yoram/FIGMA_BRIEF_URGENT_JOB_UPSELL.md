# Figma 업무지시서 — 급구 유도(업셀) 화면

**프로젝트:** HairSpare (미용실 스페어 급구 플랫폼)  
**대상 화면:** 샵(미용실) 공고 등록 → 「공고 등록하기」 직후 노출되는 **급구 유도 페이지**  
**플랫폼:** iOS / Android 모바일 (375×812 기준, iPhone 14 Pro 세이프에어어 반영)  
**작성일:** 2026-05-29  
**참고 코드:** `lib/screens/shop/job_urgent_upsell_screen.dart`, `lib/widgets/shop_job_new/shop_job_urgent_upsell_content.dart`

---

## 1. 배경·목적

### 사용자 플로우
```
공고 등록 폼 작성 → 「공고 등록하기」 탭
  → (유효성 통과 후) 급구 유도 화면 진입  ← ★ 이번 디자인 대상
    → 「급구로 등록하기」 → 급구 공고 등록 완료
    → 「일반 공고로 등록하기」 → 일반 공고 등록 완료
    → 「돌아가서 수정하기」 → 폼으로 복귀
```

### 디자인 목표
- **이벤트·프로모션·광고 랜딩** 느낌 — 안내 카드 나열이 아닌 “와, 이런 게 있구나!” 하는 첫인상
- **가시성·가독성** — 3초 안에 “급구 = 빠른 채용 + 우선 노출”이 전달
- **브랜드 일관성** — 앱 전체 Purple 기반 디자인 시스템과 충돌하지 않되, 급구만 **Red/Orange** 강조 팔레트 사용
- **세이프에어어 준수** — 히어로 그라데이션은 상단까지 풀블리드 가능하나, **텍스트·뱃지는 노치/시계와 겹치지 않음**

### 현재 구현의 문제 (개선 요청)
| 항목 | 문제 |
|------|------|
| 레이아웃 | 히어로 뱃지·헤드라인이 iOS 상태바(시계)와 겹침 |
| 톤 | 단색 그라데이션 + 텍스트만 있어 “이벤트”감 부족 |
| 계층 | 혜택·비교·유스케이스가 평면적, 시선 유도 약함 |
| CTA | 하단 버튼은 있으나 상단 히어로와 시각적 스토리 연결 약함 |

## 3. 화면 스펙 — 급구 유도 페이지

### 3.1 프레임
- **375 × 812** (또는 390 × 844) + Safe Area top/bottom 표시
- 배경: `#F9FAFB`
- 스크롤 가능 단일 컬럼 + **하단 고정 CTA 영역** (White + 상단 shadow)

### 3.2 상단 히어로 (필수 — 이벤트감 핵심)

**레이아웃 규칙**
- 그라데이션 배경: 화면 최상단까지 풀블리드 OK
- **콘텐츠 시작 Y:** `Safe Area Top + 56px(AppBar) + 8px` 이하로 올리지 말 것
- 뒤로가기: White 아이콘, 투명 AppBar (그라데이션 위)
- 히어로 하단: `radius-2xl` (24px) 둥근 모서리

**포함 요소 (카피는 유지·다듬기 가능)**
| 요소 | 현재 카피 |
|------|-----------|
| 이벤트 뱃지 | ⚡ 급구 채용 이벤트 |
| 헤드라인 | 오늘 필요한 인력, / 오늘 채우세요 |
| 서브카피 | 스페어 앱 홈·공고 목록 최상단에 급구 배지와 함께 노출됩니다. |

**디자인 방향 (Figma에서 자유 제안)**
- 번개·시계·사람 실루엣 등 **이벤트 일러스트** 또는 **글래스모피즘 카드** 오버레이
- 미세 **노이즈/패턴** 또는 **라이트 스트릭**으로 광고 랜딩 느낌
- 헤드라인 한 줄 강조(예: “오늘 채우세요”만 더 크게)
- Purple 브랜드와 연결: 히어로 아래 작은 “HairSpare for Shop” 또는 로고 워터마크 (선택)

### 3.3 혜택 3칸 그리드

| 칸 | 아이콘 | 타이틀 | 서브 |
|----|--------|--------|------|
| 1 | visibility | 우선 노출 | 홈·목록 상단 |
| 2 | bolt | 급구 배지 | 눈에 띄는 강조 |
| 3 | people | 빠른 지원 | 시급한 채용에 |

- 카드: White, border `#E5E7EB`, radius-lg, shadow-sm
- 아이콘 원형 배경: `#FEE2E2`, 아이콘 `#EF4444`

### 3.4 비교 카드 — 「급구 vs 일반 공고」

| | 일반 | 급구 (하이라이트) |
|--|------|-------------------|
| 배경 | `#F9FAFB` | `#FEE2E2` |
| 테두리 | `#E5E7EB` | `#EF4444` 25% |
| 항목 | 기본 노출, 일반 배지 | 홈·목록 우선, 급구 배지, 지원 유입 ↑ |

- 급구 열이 시각적으로 **승자**처럼 보이게 (크기·색·체크 아이콘)

### 3.5 유스케이스 블록

- 배경: `#FFF7ED`, 테두리 `#FFEDD5`, radius-xl
- 제목: 「이럴 때 급구를 쓰세요」
- 리스트 3줄 (이모지 또는 아이콘):
  - 오늘·내일 바로 필요한 스텝/디자이너
  - 노쇼·결원으로 급히 메워야 할 때
  - 일반 공고보다 빠른 지원이 필요할 때

### 3.6 하단 고정 CTA

| 순서 | 스타일 | 라벨 |
|------|--------|------|
| 1 | 그라데이션 Primary (높이 52px) + ⚡ 아이콘 | **급구로 등록하기** |
| 2 | Outlined White (높이 48px) | 일반 공고로 등록하기 |
| 3 | Text Button | 돌아가서 수정하기 |

- CTA 바: White 배경, 상단 shadow (blur 12, y -4, black 6%)
- Safe Area bottom 반영

## 6. 개발 핸드오프 메모

- Flutter 구현 시 `AppTheme` 상수와 1:1 HEX 매칭
- 히어로 top padding: `MediaQuery.padding.top + kToolbarHeight + 8`
- CTA 동작·카피는 코드와 동일 유지 (기능 변경 없음, **UI만**)
- 하단 샵 탭바는 이 화면에서 **숨김** (풀스크린 모달 플로우) — Figma에 탭바 없이 설계
