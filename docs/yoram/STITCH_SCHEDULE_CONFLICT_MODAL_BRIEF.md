> **사용 방법:** 이 파일 전체 + **현재 화면 스크린샷 1장**을 Stitch `Redesign` 모드에 붙여넣기

## 1. 이 화면이 뭔가요?

스페어가 **같은 날 시간이 겹치는 기존 근무**가 있는 상태에서 새 행동을 하려 할 때 뜨는 **바텀 시트 모달**입니다.

| 진입 경로 | `actionLabel` (동적 문구) |
|-----------|---------------------------|
| 공고 상세 → 지원하기 | `지원` |
| 공고 상세 / 제안 리뷰 → 제안 수락 | `제안 수락` |

**사용자 목표:** 겹치는 일정을 정리(취소)하거나, 스케줄표에서 확인한 뒤 원래 행동(지원·수락)을 이어감.

## 3. 현재 카피 (한국어 — 의미 유지 필수)

### 헤더
| 요소 | 문구 |
|------|------|
| 제목 | `근무 시간이 겹쳐요` ※코드 오타 `겹려요` → 수정 예정 |
| 부제 | `같은 날 다른 근무와 {actionLabel} 시간이 겹칩니다. 아래 일정을 정리한 뒤 진행할 수 있어요.` |
| 닫기 | X 버튼 (우상단) |

### 오렌지 안내 배너 (`_OverlapHintBanner`)
`하루에 시간이 겹치는 근무는 등록할 수 없어요. 취소 가능한 기존 근무를 정리하면 {actionLabel}을(를) 이어서 진행할 수 있습니다.`

### 취소 정책 요약 (`_CancellationPolicySummaryCard`) — 상위 3줄만 표시
1. `확정된 근무를 취소하면 예약 에너지는 환불되지 않습니다.`
2. `취소 시 해당 매장·스페어 채팅방에 자동으로 알림이 전송됩니다.`
3. `근무 시작 시각 이후에는 앱에서 취소할 수 없습니다.`

### 빨간 배너 (`_BlockedContactBanner`) — **모든 겹침 일정이 앱에서 취소 불가일 때만**
`겹치는 근무를 앱에서 취소할 수 없습니다. 근무 시작 후이거나 확정되지 않은 일정일 수 있습니다. 스케줄표에서 확인해 주세요.`

### 섹션 제목
`겹치는 근무`

### 겹침 일정 카드 (`_ConflictScheduleCard`) — 카드당
| 요소 | 내용 |
|------|------|
| 칩 1 | `겹침` (빨강 배경, 흰 글자) |
| 칩 2 | 취소 가능 여부 — `취소 가능` / `D-1 이내 취소 불가` / `근무 시작 후` / `완료된 근무` / `취소됨` / **`확정 전`** 등 |
| 칩 3 | `제안 대기` (보라) 또는 `근무 예정` (파랑) |
| 본문 | 매장명 (굵게) |
| 메타 | 📅 `M월 d일 (E)` · 🕐 `HH:mm~HH:mm` |

### 하단 CTA

**상태 A — 취소 가능한 겹침이 1건 이상**
| 버튼 | 문구 |
|------|------|
| Primary (filled) | `겹치는 근무 취소 후 계속` 또는 `겹치는 근무 N건 취소 후 계속` |
| Outlined | `스케줄표에서 확인` |
| Text | `나중에 할게요` |

## 5. UX·가독성 이슈 (리디자인 이유)

1. **정보 박스가 3겹** — 오렌지 안내 + 회색 정책 + (조건부) 빨간 배너가 연속으로 나와 **스캔이 어렵다**
2. **글래스모피즘 과다** — 반투명 카드·배너가 겹쳐 뒤 배경과 섞여 **텍스트 대비가 낮다**
3. **색이 너무 많다** — 오렌지·빨강·보라·파랑·인디고·초록(취소 가능 칩)이 한 화면에 공존
4. **Primary CTA 색이 앱 브랜드(보라)와 다름** — 홈·하단탭과 이질감
5. **칩 3개가 한 줄** — `겹침` + `확정 전` + `제안 대기` 등 **밀도가 높아** 매장명까지 내려가야 함
6. **취소 불가 시** Primary 자리 회색 문구 + 빨간 배너 + 카드 내 칩이 **중복 안내**처럼 느껴짐

## 6. 유지해야 할 것 (변경 금지)

- 바텀 시트 형태 (풀스크린 페이지 X)
- 버튼 3종 역할: **취소 후 계속** / **스케줄표** / **나중에**
- `actionLabel` 동적 삽입 (`지원`, `제안 수락`)
- 겹침 일정 **리스트** (0건이면 모달 자체가 안 뜸)
- 취소 정책 bullet **법무·운영 문구** (요약 가능, 의미 변경 X)
- 취소 가능/불가 **상태별 UI 분기** (Primary vs 안내 문구)
- 스크롤 가능 (겹침 2건 이상)

## 7. Stitch 디자인 방향 제안 (자유롭게 개선 OK)

- **홈 Stitch 톤:** white 카드 + `stitchSoftShadow` + 16px radius, CTA `#9333EA`
- 정책·안내는 **아코디언 1개** 또는 **아이콘+한 줄 요약 + 「자세히」** 로 압축 검토
- 위험(취소 불가)은 **빨강 전면 배너 1개**로 통합, 오렌지 박스 제거 또는 병합
- 겹침 카드: `StitchListJobCard` / 스케줄 카드와 **같은 카드 언어**
- 글래스: **절제** — 패널 배경만 살짝 blur, 내부 카드는 **불투명 white**
- 칩: 최대 2개 노출 또는 상태를 **한 줄 status line**으로 단순화
- 히어로: ⏱️ 이모지 대신 **outline 아이콘 + 보라 원** (홈 카테고리 그리드와 통일)
