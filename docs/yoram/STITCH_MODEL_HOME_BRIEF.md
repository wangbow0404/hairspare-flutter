> **사용 방법:** 이 파일 전체를 Stitch 캔버스 프롬프트에 복사·붙여넣기 하세요.  
> **플랫폼:** 모바일 iOS/Android (390×844), 라이트 테마, 한국어 UI  
> **첨부 권장:** 현재 모델 홈 스크린샷(급구 배너·결제 탭 잔존 버전) — **교체 대상**으로 표시  
> **관련 문서:** `STITCH_SPARE_SIGNUP_BRIEF.md` (모델 가입 폼), 스페어 홈·모델매칭 스와이프 화면

## 1. 제품 맥락

### 1.1 HairSpare 역할 구조

- **HairSpare(헤어스페어)** — 미용 인력·헤어 모델 매칭 플랫폼
- **스페어·디자이너** — 일자리 탐색·공고 지원·스케줄·에너지 결제·**모델매칭(스와이프로 모델 찾기)**
- **모델** — 디자이너 매칭에 **노출되는 수신자** · 채팅으로 일정 조율 · 확정 시술 스케줄

### 1.2 모델의 핵심 사용자 루프

```
프로필 완성 → 본인인증 → 매칭 노출 ON
  → 디자이너 관심/요청 수신 → 채팅 → 시술 일정 확정
```

모델은 **공고를 찾거나 지원하지 않습니다.**  
모델매칭 스와이프(디자이너가 모델을 고르는 화면)도 **모델 계정에서는 사용하지 않습니다.**

### 1.3 현재 문제 (첨부 스크린샷 기준)

| 요소 | 문제 |
|------|------|
| 급구 히어로 배너 | 스페어 구직 CTA — 모델과 무관 |
| 결제·찜 하단 탭 | 모델 여정에 없음 |
| 프로필 배너만 추가 | 스페어 홈에 얹은 임시 패치 느낌 |
| 매칭 요청·채팅·일정 | 홈에서 한눈에 안 보임 |
| 본인인증 필수 | 홈에서 인증 상태가 약함 |

**→ 모델 홈은 스페어 홈과 IA·탭·카피를 완전히 분리해 새로 디자인합니다.**

## 2. 디자인 원칙

### 2.1 한 줄 정의

| | 정의 |
|---|------|
| **스페어 홈** | "일자리 찾기" — 공고·급구·지원 중심 |
| **모델 홈** | "매칭 대기실" — 프로필·수신·일정·채팅 중심 |

### 2.2 공유 vs 분리

**같은 앱에서 공유:**
- HairSpare 브랜드·로고
- 본인인증 플로우
- 채팅·메시지 인프라
- Stitch 보라 디자인 토큰

**완전히 분리:**
- 홈 첫 화면 구성
- 하단 탭 메뉴
- 히어로/배너 영역
- 카피·empty state
- 검색·CTA 목적

### 2.3 벤치마크 (참고만, 복제 금지)

| 앱 | 모델에 적용할 점 |

| Tinder/Bumble (수신측) | 프로필·매칭·채팅 중심, 탐색 최소 |
| StyleSeat | 고객 vs 프로 역할별 다른 홈 |
| 숨고/크몽 | 동일 앱 셸, 역할별 메뉴 노출/비노출 |

## 3. 정보 구조 (IA)

### 3.1 모델 홈 — 스크롤 구성 (위 → 아래)

```
┌─────────────────────────────────┐
│ AppBar: 로고 · 알림 · 메시지(뱃지) │  ← 검색 제거 권장
├─────────────────────────────────┤
│ [내 프로필 카드]                  │  ← 히어로 대체 (급구 배너 X)
│  사진·이름·지역·기장·한줄소개      │
│  완성도 % · 매칭 노출 토글         │
│  [프로필 편집] [카드 미리보기]     │
├─────────────────────────────────┤
│ [상태 스트립]                     │
│  본인인증 · 오늘 관심 N건 · 노출상태│
├─────────────────────────────────┤
│ 받은 관심 / 매칭 요청              │  ← 섹션 1 (최우선)
├─────────────────────────────────┤
│ 다가오는 시술 일정                 │  ← 섹션 2
├─────────────────────────────────┤
│ 최근 메시지                        │  ← 섹션 3
├─────────────────────────────────┤
│ 빠른 메뉴 (4개 이하)               │
├─────────────────────────────────┤
│ 고객센터 · 약관 (축소)             │
└─────────────────────────────────┘
│ 홈 │ 메시지 │ 스케줄 │ 마이        │  ← 하단 탭
└─────────────────────────────────┘
```

### 3.2 스페어 홈 vs 모델 홈 비교

| 영역 | 스페어·디자이너 | 모델 |
|------|----------------|------|
| 히어로 | 급구·에너지·교육 배너 | **내 프로필 카드** |
| 본문 | 급구/인기/신규 공고 섹션 | **받은 관심 · 일정 · 메시지** |
| 카테고리 | 공고별·스케줄·포인트·공간대여·모델매칭 등 | **스케줄·메시지·프로필·교육** (4개 이하) |
| 하단 탭 | 홈 · 결제 · 찜 · 마이 | **홈 · 메시지 · 스케줄 · 마이** |
| 검색 | O | **X** (또는 프로필 미리보기 아이콘) |

### 3.3 하단 탭 — 모델 전용

| 탭 | 역할 | 아이콘 제안 |
|----|------|-------------|
| **홈** | 대시보드 (본 문서) | home |
| **메시지** | 매칭 후 채팅 허브 | chat |
| **스케줄** | 확정된 시술 일정 | calendar |
| **마이** | 프로필·본인인증·설정 | person |

**제거:** 결제, 찜

## 4. 화면별 상세 스펙

### 4.1 내 프로필 카드 (히어로 대체)

**목적:** 디자이너에게 보이는 내 카드 + 노출 제어

| 요소 | 스펙 |
|------|------|
| 대표 사진 | 1:1 또는 4:5, 매칭 카드와 동일 비율 |
| 이름 | 가입 시 입력명 |
| 메타 | 활동 지역 · 현재 기장 |
| 한줄 소개 | 1줄 말줄임 |
| 완성도 | % + 프로그레스 바 |
| 미완성 항목 | 칩: 사진 / 기장 / 소개 / 본인인증 |
| 매칭 노출 | ON/OFF 토글 (보라) |
| CTA | `프로필 편집` · `카드 미리보기` |

**카피 예:**
- 타이틀: "모델테스트님, 모델 프로필"
- 서브: "디자이너에게 보이는 내 카드예요"
- 노출 OFF: "지금은 디자이너에게 보이지 않아요"

**프로필 필드 (가입·편집과 정합):**
- 생년월일, 성별, 활동 지역(전국 시/도·구/군), 현재 기장, 선호 시술, 모델 이미지 태그, 모델 경력, 한줄 소개, 프로필 사진 1~3장
- 촬영은 **얼굴 공개 필수** (촬영 협의 선택 UI 없음)

---

### 4.2 상태 스트립

히어로 바로 아래, 얇은 가로 영역 (1줄 또는 가로 스크롤 칩)

| 상태 | 표시 |
|------|------|
| 본인인증 미완료 | 🔒 본인인증 필요 → `인증하기` |
| 본인인증 완료 | ✓ 인증 완료 |
| 오늘 관심 | "오늘 받은 관심 **3**건" (숫자 강조) |
| 매칭 노출 OFF | "매칭 노출 꺼짐" |

---

### 4.3 받은 관심 / 매칭 요청 (섹션 1)

**목적:** 디자이너가 모델에게 관심(하트)을 보낸 경우 모델이 확인·응답

**카드 1장 구성:**
- 디자이너 프로필 사진(또는 이니셜)
- 이름 · 활동 지역
- 관심 시술 유형 (예: 전체염색, 레이어드컷)
- 시간 (예: 2시간 전)
- CTA: `채팅하기` 또는 `수락`

**섹션 헤더:** "받은 관심" + `전체보기`

**Empty state:**
- 일러스트 또는 심플 아이콘
- "아직 받은 관심이 없어요"
- "프로필을 완성하고 매칭 노출을 켜면 디자이너가 찾아올 거예요"

---

### 4.4 다가오는 시술 일정 (섹션 2)

**카드 1장:**
- 날짜·시간
- 시술 종류
- 샵/디자이너 이름
- 상태: 확정 / 조율 중

**Empty:** "예정된 시술이 없어요"

---

### 4.5 최근 메시지 (섹션 3)

- 최대 2건 프리뷰
- 상대 이름 · 마지막 메시지 말줄임 · 시간 · 읽지 않음 뱃지
- `메시지 전체보기` → 메시지 탭

---

### 4.6 빠른 메뉴

4개 이하, 원형 아이콘 + 라벨 (스페어 홈 CategoryGrid 톤 유지)

| 메뉴 | 비고 |
|------|------|
| 스케줄표 | WorkCheck / 일정 |
| 메시지 | 채팅 목록 |
| 프로필 편집 | 모델 프로필 |
| 교육 | 선택 (2차) |

**포함하지 않음:** 공고별, +포인트, 공간대여, 모델매칭(스와이프), 결제

---

### 4.7 프로필 카드 미리보기 (별도 화면 #6)

디자이너 **모델매칭 스와이프 카드**와 동일 레이아웃으로 풀스크린 미리보기

표시 필드:
- 대표 사진 (스와이프 가능 1~3장)
- 이름 · 나이(생년월일 기반) · 성별
- 지역 · 기장
- 선호 시술 · 모델 이미지 태그
- 경력 · 한줄 소개

하단: `프로필 편집하기` CTA

## 5. 상태 변형 (Stitch 필수)

### 5.1 본인인증 전 (#3)

- 홈 콘텐츠 **반투명 dim** 또는 블러
- 하단 **고정 CTA:** `본인인증 하기` (보라, full width)
- 카피: "모델 매칭 노출을 위해 본인인증이 필요해요"
- **나중에 하기 / 홈 건너뛰기 없음**

### 5.2 프로필 미완성 (#4)

- 완성도 70% 미만 시 체크리스트 카드 강조
- 항목별 체크: □ 사진 □ 기장 □ 한줄소개 □ 본인인증
- 각 항목 탭 → 편집 화면 (와이어 수준 링크 표시만)

### 5.3 매칭 노출 OFF (#5)

- 프로필 카드에 "노출 꺼짐" 배지
- 본문 empty 톤: "매칭 노출을 켜면 디자이너에게 보여요"
- 토글 ON 유도

### 5.4 Active vs Empty (#1, #2)

| | Active | Empty |
|---|--------|-------|
| 받은 관심 | 2~3 카드 | empty illustration |
| 일정 | 1~2 카드 | "예정 없음" |
| 메시지 | 1~2 프리뷰 | "대화 없음" |

## 8. 카피 가이드 (한국어)

### 8.1 톤
- 친근하지만 전문적
- 모델 = "피시술자·매칭 수신자" 포지션
- "구직" "지원" "급구" 단어 **사용 금지**

### 8.2 권장 문구

| 상황 | 문구 |
|------|------|
| 프로필 카드 서브 | 디자이너에게 보이는 내 카드예요 |
| 완성도 | 프로필 완성도 |
| 노출 ON | 매칭 노출 |
| 노출 OFF | 지금은 디자이너에게 보이지 않아요 |
| 인증 전 | 모델 매칭 노출을 위해 본인인증이 필요해요 |
| 관심 empty | 아직 받은 관심이 없어요 |
| 일정 empty | 예정된 시술이 없어요 |

## 9. 스페어 홈과의 관계 (구현 참고 — Stitch는 무시 가능)

- 동일 Flutter 앱, `spareSubtype: model`일 때 **별도 홈 + 별도 탭** 라우팅
- 스페어 홈(`SpareHomeScrollView`)은 변경 최소화
- 모델 홈은 **신규 화면**으로 설계 (이 브리프 기준)

## 10. 의사결정 메모 (팀 확인용)

Stitch 작업 전 확정하면 좋은 항목:

1. **「받은 관심」vs「채팅 시작됨」** — 같은 리스트에 둘지, 탭으로 나눌지
2. **검색** — 모델 AppBar에서 완전 제거 vs 프로필 미리보기 아이콘으로 대체
3. **교육·챌린지·스토어** — 1차 홈 빠른메뉴 포함 여부 (권장: 교육만 선택)
