# Stitch 업무지시서 — 관리자 「비즈니스 설정」(M15) 콘텐츠 명세

> **용도:** Stitch AI에 비즈니스 설정 화면 **재디자인을 요청할 때** 이 파일을 붙여넣습니다.  
> **범위:** 화면에 **무엇이 들어가야 하는지**(정보 구조·항목·동작·비즈니스 맥락)만 기술합니다. **색상·레이아웃·컴포넌트 스타일 등 디자인 지침은 본 문서에 포함하지 않습니다.**  
> **모듈 ID:** M15  
> **Flutter 경로:** `lib/screens/admin/admin_settings_screen.dart`  
> **API:** `GET /api/admin/settings`, `PATCH /api/admin/settings/{key}`  
> **관련 문서:** `ADMIN_PANEL_SPECS_APPENDIX.md` §B, `ADMIN_PANEL_FEATURE_PLAN.md` §M15

---

## 1. 화면 목적

HairSpare 운영자(관리자)가 **앱 전역에 적용되는 비즈니스 규칙**을 서버에 저장·변경하는 화면입니다.

- 결제·수수료(에너지, 급구, **하이패스**, 구독 등)
- 역할·등급별 **한도**(매칭, 공고 수, 에너지 구매)
- **제재 정책**(연락처 유출, 취소, 노쇼)
- 공고 **노출·인기도** 가중치
- **공간대여** 예약 규칙

변경 시 **감사 로그**에 기록되며, 운영 책임 추적의 기준이 됩니다.

## 3. 화면 정보 구조 (기능)

### 3.1 그룹 탭 (5개)

설정은 **상호 배타적인 그룹 탭**으로 나뉩니다. 한 번에 하나의 그룹만 편집 영역에 표시합니다.

| 탭 ID | 탭 이름 | 그룹 설명 (탭 아래 안내 문구) |
|-------|---------|------------------------------|
| `pricing` | 경제·가격 | 에너지·구독·급구·하이패스 수수료 |
| `quota` | 쿼터·한도 | 매칭·에너지·등급별 공고 한도 |
| `sanction` | 제재정책 | 연락처·취소·노쇼 제재 규칙 |
| `ranking` | 랭킹·노출 | 공고 인기도·노출 가중치 |
| `space` | 공간대여 | 공간 예약 시간·기간 규칙 |

### 3.2 그룹 내 항목 공통 형태

각 설정 항목은 최소 다음 정보를 가집니다.

| 필드 | 설명 |
|------|------|
| `key` | 서버 저장 키 (개발용, UI에 노출 불필요) |
| `label` | 관리자에게 보이는 한글 라벨 |
| `type` | 현재는 전부 `int` (정수) |
| `value` | 현재 서버에 저장된 값 |
| `min` | 저장 시 허용 최소값 |

**입력 단위 구분 (콘텐츠 규칙):**

- **원화(KRW):** 라벨에 `(원)` 포함, 또는 `Fee`·`Cost`·`Deposit`·`Price` 계열 키
- **개수·횟수·시간:** `(개)`, `(회)`, `(일)`, `(시간)`, `(시)` 등 라벨에 단위 명시
- `maxEnergyPurchaseAmount`는 **에너지 개수**이지 원화가 아님 (₩ 표시 금지)

### 3.3 화면 동작 (필수)

| 동작 | 설명 |
|------|------|
| 초기 로드 | 서버에서 그룹·항목·현재값 조회. 실패 시 재시도 가능 상태 |
| 일괄 편집 | 그룹 내 여러 필드를 수정한 뒤 **한 번에 저장** |
| 저장 전 확인 | 변경 건수 안내 + 「감사 로그에 기록됩니다」 확인 |
| 저장 검증 | 모든 변경값은 정수이며, 각 항목 `min` 이상이어야 함 |
| 되돌리기 | 마지막 **서버에서 불러온 값**으로 폼 복원 (저장 전 로컬 변경 취소) |
| 저장 후 | 서버에 즉시 반영. 성공 메시지 표시 |

### 3.4 상태

| 상태 | 처리 |
|------|------|
| 로딩 | 설정 조회 중 |
| 오류 | 조회 실패 메시지 + 다시 시도 |
| 저장 중 | 저장 버튼 비활성·진행 표시 |
| 변경 없음 | 저장 시 「변경된 항목이 없습니다」 |

## 4. 그룹별 설정 항목 — 현재 구현 (30개)

> 아래는 **지금 서버·앱에 실제로 존재하는 항목**입니다. Stitch 재디자인 시 **이 30개 + §5 후속 항목**을 모두 수용할 수 있는 정보 구조를 전제로 합니다.

### 4.1 경제·가격 (`pricing`) — 8개

| 라벨 | key | 기본값 | min | 단위 | 비즈니스 의미·영향 범위 |
|------|-----|--------|-----|------|-------------------------|
| 에너지 1개당 가격 (원) | `energyPointCostPerUnit` | 1000 | 1 | 원 | 포인트로 에너지 구매 시 1에너지당 차감 포인트. 스페어 에너지 결제 |
| 급구 공고 수수료 (원) | `urgentJobListingFee` | 5000 | 0 | 원 | 샵이 공고를 **급구**로 등록할 때 부과. 급구 결제 화면 |
| **하이패스 노출 수수료 (원)** | `hipassListingFee` | 5000 | 0 | 원 | 샵 **첫 공고** 등록 후 **하이패스** 프리미엄 노출 구역 결제. `job_opening_soon_upsell_screen` |
| 월 구독료 (원) | `subscriptionMonthlyFee` | 99000 | 0 | 원 | 샵 월 구독 플랜 가격 (결제·구독 화면) |
| 프리미엄 공고 수수료 (원) | `premiumJobFee` | 5000 | 0 | 원 | 프리미엄 매장 공고 부가 수수료 |
| 채팅 애드온 수수료 (원) | `chatAddonFee` | 2000 | 0 | 원 | 구독 외 채팅 기능 추가 구매 |
| 모델 보증금 (원) | `modelDepositAmount` | 30000 | 0 | 원 | 모델 매칭·시술 예약 시 보증금 기준 |
| 공고 에너지 환산 (원÷N=에너지) | `jobEnergyFormulaDivisor` | 1000 | 1 | 환산 N | 공고 일당 금액을 에너지 개수로 변환: `amount ÷ N`. 공고 등록 |

**하이패스 맥락 (Stitch에 전달할 도메인 설명):**

- **하이패스(HIPASS)** = 샵이 **첫 번째 공고**를 등록했을 때만 제안되는 **프리미엄 노출** 유료 옵션
- 홈 화면 **하이패스 구역**에 노출, 급구와 별도 상품
- 관리자는 `hipassListingFee`로 수수료 금액만 조정 (노출 조건·UI 카피는 앱 로직)

### 4.3 제재정책 (`sanction`) — 6개

| 라벨 | key | 기본값 | min | 단위 | 비즈니스 의미·영향 범위 |
|------|-----|--------|-----|------|-------------------------|
| 채팅당 연락처 시도 최대 횟수 | `contactMaxAttemptsPerChat` | 3 | 1 | 회 | 채팅에서 연락처(전화·카톡 등) 전송 시도 누적 한도. 초과 시 대화방 삭제·지원 취소 등 |
| 미용실 연락처 위반 정지 (일) | `shopContactPenaltyDays` | 1 | 0 | 일 | 샵이 연락처 유출 적발 시 대화·공고 제한 기간 |
| 샵 누적 제재 시 탈퇴 기준 (회) | `maxShopRoomPenaltiesBeforeBan` | 3 | 1 | 회 | 동일 사업자·샵이 연락처 제재를 N회 받으면 계정 탈퇴·블랙리스트 |
| 30일 내 일방 취소 허용 횟수 | `shopUnilateralCancelLimit30d` | 3 | 0 | 회 | 샵이 확정 근무를 일방 취소할 수 있는 30일 누적 횟수. 초과 시 공고 등록 정지 |
| 공고 등록 정지 기간 (일) | `shopJobPostingSuspensionDays` | 7 | 0 | 일 | 위 취소 한도 초과 시 신규 공고 등록 불가 기간 |
| 늦은 취소·노쇼 기준 (시간) | `lateCancelCutoffHours` | 48 | 1 | 시간 | 근무 시작 N시간 이내 취소 = 노쇼·위약금 동일 처리 기준 |

**스페어 측:** 3회 연락처 시도 시 지원 취소·잠금 에너지 몰수 (정책은 코드에 있으나 별도 토글 항목은 §5.3)

### 4.4 랭킹·노출 (`ranking`) — 6개

| 라벨 | key | 기본값 | min | 단위 | 비즈니스 의미·영향 범위 |
|------|-----|--------|-----|------|-------------------------|
| 인기 공고 상위 N개 | `jobPopularityTopN` | 10 | 1 | 개 | 스페어 홈 등에 「인기」 배지 붙는 공고 수 |
| 신규 공고 보너스 시간 (시간) | `newJobBonusWindowHours` | 72 | 1 | 시간 | 등록 후 N시간 동안 인기도 점수 보너스 |
| 인기도 — 지원 수 가중치 | `jobPopularityAppWeight` | 10 | 0 | 배수 | 지원 1건당 점수 = 지원수 × 이 값 |
| 인기도 — 조회 수 가중치 | `jobPopularityViewWeight` | 1 | 0 | 배수 | 조회 1건당 점수 |
| 인기도 — 프리미엄 보너스 | `jobPopularityPremiumBonus` | 5 | 0 | 점 | 프리미엄 매장 공고 가산점 |
| 인기도 — 저에너지 보너스 | `jobPopularityLowEnergyBonus` | 2 | 0 | 점 | 에너지 3개 이하 공고 가산점 (장벽 낮음) |

**참고:** 급구 공고는 「급구」 배지가 우선이라 인기 배지와 중복 없음. 인기 선정 시 **샵당 1개 공고만** 상위에 올라가도록 앱 로직에 있음.

## 7. API·데이터 계약 (개발 연동)

### 7.1 조회

```
GET /api/admin/settings
→ { "data": { "groups": [ { "id", "title", "description", "settings": [ { "key", "label", "type", "value", "min" } ] } ] } }
```

### 7.2 변경

```
PATCH /api/admin/settings/{key}
Body: { "value": <int> }
```

- 없는 key → 404  
- `value < min` → 400  
- 성공 시 `AdminAuditLog`에 `update_config` 기록  

### 7.3 감사 로그 (관리자 안내 문구)

저장 확인 시 반드시 전달할 문구 예:

> 「N개 항목을 저장합니다. 감사 로그에 기록됩니다.」  
> 「저장 시 서버에 즉시 반영됩니다.」
