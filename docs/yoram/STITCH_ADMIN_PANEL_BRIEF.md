## 0. 글로벌 디자인 규칙 (모든 화면 공통)

### 0.1 레이아웃 골격 (기존 `AdminLayout` 일치 유지)

```
┌──────────────────────────────────────────────┐
│ 상단 헤더 (높이 80 데스크톱 / 56 모바일)        │
│  로고+「관리자」  ·  실시간 업데이트중  ·  로그아웃 │
├────────────┬─────────────────────────────────┤
│ 사이드바    │ 본문 (스크롤)                     │
│ 288px      │  페이지헤더 + 필터바 + 테이블/카드  │
│ (그룹 메뉴) │                                  │
└────────────┴─────────────────────────────────┘
```

- **데스크톱(≥768):** 사이드바 고정 노출(288px), 본문 우측.
- **모바일(<768):** 사이드바 → 햄버거 drawer, 헤더 56px.
- 배경: `adminBackgroundGradient`(purple50→blue50→pink50 대각). 카드: White + radius 12 + soft shadow.

### 0.2 색상 토큰 (기존 `app_theme.dart` 준수)

| 용도 | 값 |
|------|-----|
| 활성 메뉴/주요 강조 그라데이션 | `primaryPurple500 → primaryPink` (좌→우) |
| 사이드바 보더 | `adminPurple100 #EDE9FE` |
| 배경 그라데 | `adminPurple50 #F5F3FF` / `adminPink50 #FDF2F8` |
| 본문 텍스트 | `textPrimary #111827` / `textSecondary #6B7280` / `textTertiary #9CA3AF` |
| 상태색 | 성공 green / 경고 amber / 위험·제재 `urgentRed #EF4444` / 정보 blue |
| 대기성(액션 필요) | red 점/뱃지 강조 |

### 0.3 컴포넌트 4종 패턴 (모듈마다 재사용)

1. **목록 페이지** = 페이지헤더(타이틀+서브+우측 액션) + **필터바**(검색 + 상태/타입 드롭다운 + 날짜) + **데이터테이블/카드리스트**(데스크톱 테이블, 모바일 카드) + 페이지네이션.
2. **상세 페이지** = 브레드크럼/뒤로 + 요약 헤더(상태 뱃지) + **세그먼트 정보 블록** + **액션 버튼 영역**(우측 고정 또는 하단).
3. **액션 모달** = 제재/환불/승인 — 제목 + 대상 요약 + **사유 입력(필수)** + 위험액션 red 버튼 + 취소.
4. **설정 폼** = 그룹 탭 + 입력 필드(현재값 표시) + 저장/되돌리기 + "서버 반영" 안내.

### 0.4 금지 사항

다크테마, 제네릭 AI 보라 그라데이션 남발, 영문 UI, 해외 스톡 사진, 과한 이모지, 모바일 전용 UI(이건 데스크톱 우선).

## 3. 화면별 상세 스펙

### M0 — 대시보드

- **KPI 카드 그리드**(데스크톱 3열/태블릿 2열/모바일 1열): 총 회원·활성 공고·오늘 결제액·오늘 체크인·에너지 지갑·노쇼 — **+ 신규**: 대기 인증(red)·미처리 신고(red)·대기 공간예약·대기 교육승인·오늘 모델매칭.
- 카드: 좌측 타이틀+값+증감, 우측 아이콘 칩(컬러). 탭 → 해당 모듈.
- **역할별 회원** 블록(스페어/미용실/모델·판매자) 가로 카드.
- **최근 활동** 피드(점 컬러 + 라벨·대상 + 시각), "전체보기" → 감사로그(M18).
- 헤더 우측 "실시간 업데이트중"(green dot, 기존 유지).

### M1 — 회원 관리

- **목록:** 필터바(검색·역할·가입수단) + 테이블(이름·이메일·역할뱃지·가입일·상태·`_count` 요약·관리버튼).
- **상세:** 상단 요약(아바타·이름·역할·상태뱃지) + 탭[기본정보 / 활동 / 지갑(에너지·포인트) / 제재이력 / 인증상태] + 우측 액션 패널(정지·역할변경·에너지조정·포인트조정·강제탈퇴 — 위험은 red).
- **제재모달:** 대상 요약 + 기간 선택 + 사유(필수) + 확인.

### M2 — 인증 심사 큐

- **목록:** 탭[사업자/신원/대리인] + 상태필터(대기 우선·상단 정렬) + 카드(신청자·제출일·상태).
- **상세:** 2열 — 좌측 제출정보(사업자번호·상호·대표·주소), 우측 **증빙 뷰어**(사업자등록증 이미지 + OCR 추출값 + NTS 검증결과 일치/불일치 하이라이트). 하단 승인(green)/반려(red+사유) 버튼.

### M3 — 공고 관리

- **목록:** 필터(상태·급구·검색) + 테이블(제목·샵·지역·금액·에너지·플래그칩[급구/프리미엄/숨김]·지원수·상태).
- **상세:** 공고 원본 미리보기(모바일 카드 프리뷰) + 지표(지원·조회) + 소유 샵 링크 + 액션(숨김·강제마감·급구/프리미엄 토글·삭제).

### M4 — 스케줄·체크인

- **목록:** 날짜필터 + 테이블(스페어·샵·공고·예정시간·체크인/아웃·상태칩).
- **개입모달:** 강제완료/강제취소/노쇼처리 — 노쇼는 대상(스페어/샵) 선택 + 사유 → M13 제재 연동 안내.

### M5 — 모델 매칭

- **목록:** 탭[좋아요/매칭] 상태필터 + 모델 프로필 그리드(썸네일·이름·지역·상태).
- **상세:** 매칭 양측 카드(디자이너↔모델) + 생성 채팅 링크 + 취소이력 + 강제 매칭취소(사유).

### M6 — 공간 대여

- **목록:** 탭[공간/예약] — 공간(상태·숨김·등록샵), 예약(대기 강조·예약자·시간·금액·상태).
- **상세:** 공간정보·운영스케줄·리뷰 + 예약 타임라인. **분쟁모달:** 결정(승인/취소/환불) + 사유.

### M7 — 교육 관리

- **목록:** 필터(온/오프·카테고리·상태) + 테이블(제목·강사·정원/신청·에너지·마감).
- **상세:** 교육 원본·커리큘럼·수강자 목록 + 액션(숨김·삭제·수강 환불[에너지 환원]).

### M8 — 결제 관리

- **목록:** 필터(타입·상태) + 테이블(주문ID·결제수단·사용자·타입칩·금액·상태·일시).
- **상세:** 결제 상세 + 연결 자원(에너지/공고). **환불모달:** 전액/부분 금액 + 사유 + 회수자원 안내(red).

### M9 — 에너지 관리

- **목록:** 필터(타입: purchase/spend/lock/forfeit) + 테이블(지갑·사용자·타입칩·증감·공고참조·상태).
- **수동지급/차감모달:** 사용자 검색 + 수량(+/-) + 사유. lock/forfeit 조정 액션.

### M10 — 포인트·미션

- **목록:** 포인트 거래(사용자·earn/spend·금액·설명) + **미션 설정폼**(미션타입·보상P·일일캡·활성토글). 어뷰징 의심 강조.

### M11 — 구독·크리에이터

- **목록:** 구독 목록 + 크리에이터 카드(구독자수·영상수·인증뱃지). 상세: 구독이력·크리에이터 인증 액션.

### M12 — 신고/제재 케이스 (핵심 워크플로)

- **목록:** 상태 칸반 또는 필터(접수/검토중/종결) + 케이스(신고유형·대상·신고자·접수일·우선도).
- **상세:** 좌측 케이스 정보·증거, 우측 **채팅 로그 감사 뷰**(연락처 위반 하이라이트) + 처리 액션(경고/정지/탈퇴 → M1·M13 연동) + 담당자 배정 + 종결(사유).

### M13 — 제재 실행·이력

- **목록:** 제재 케이스(사용자·제재유형·기간·사유·해제여부) + 블랙리스트 탭.
- **모달:** 제재 적용/해제 — 유형·기간·사유. (정책값은 M15에서 설정 — 본 화면은 실행/이력)

### M14 — 콘텐츠 모더레이션

- **목록:** 탭[영상/댓글] 신고·플래그 우선 + 썸네일/댓글 + 작성자 + 액션(숨김·삭제·피처드 토글).
- **상세:** 영상 플레이어 + 댓글 트리 + 작성자(M1).

### M15 — 비즈니스 설정

- **그룹 탭:** 경제·가격 / 쿼터·한도 / 제재정책 / 랭킹·노출 / 공간.
- 각 항목: 라벨 + **현재값 입력 필드** + 단위 + 도움말. 하단 저장/되돌리기 + "변경은 서버에 반영되며 감사로그에 기록됩니다" 안내.
- 항목 목록은 `ADMIN_PANEL_FEATURE_PLAN.md` M15 표와 1:1.

### M16 — 알림 발송

- **발송 폼:** audience(전체/역할/세그먼트) + 제목 + 본문 + 딥링크 + 미리보기 + 발송(확인모달). **템플릿 관리** + 발송 이력 목록.

### M17 — 레퍼런스 데이터

- **CRUD 목록:** 탭[지역/샵등급/매칭태그/카테고리]. 트리(지역 계층)·테이블 + 추가/수정/삭제 모달.

### M18 — 감사 로그

- **목록(타임라인/테이블):** 일시·관리자·액션칩·대상·사유 + 필터(관리자·액션·대상·기간) + before/after 펼침. 읽기 전용.

## 5. Stitch 프롬프트 사용 팁

- 모듈 단위로 본 문서의 해당 §3 절 + §0 글로벌 규칙을 함께 붙여넣기.
- 동일 컴포넌트(필터바·테이블·액션모달)는 재사용 명시 → 일관성 확보.
- 산출 후 기능 플랜(M_ ID)과 대조해 누락 화면 점검.
