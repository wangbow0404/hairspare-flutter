# HairSpare MVP Scope v1

> 기준: 현재 Flutter 코드/문서 상태 (2026-05-15 확정)  
> 목적: 이번 스프린트에서 “반드시 끝낼 것”과 “이번엔 하지 않을 것”을 고정.

## 이번 스프린트 한 문장

**Spare 핵심 플로우(공고·스케줄·근무체크·채팅/알림·챌린지)를 안정화하고, Shop은 인증·공고 등록·지원자 확인까지 포함한 뒤 staging 배포 준비까지 마친다.**

---

## 1) Product Goal

Spare/Shop 사용자가 핵심 업무(공고-스케줄-근무체크-커뮤니케이션)를 모바일에서 안정적으로 수행할 수 있게 한다.

---

## 2) Must (이번 스프린트 필수)

### Spare

1. **인증/역할 진입 안정화**
   - 역할 선택 → 로그인/세션 복원 → Spare 홈 진입
   - 서버 발급 JWT 기준 (mock은 debug/개발만)

2. **공고 핵심 플로우**
   - 공고 목록/상세/지원의 정상 동작

3. **스케줄/근무체크**
   - 스케줄 조회
   - 근무체크는 **근무 종료 시간 이후**만 가능 (시작 전·근무 중에는 안내 스낵바)
   - `ScheduleWorkSession` 기준 사용자 안내 문구 일관성

4. **채팅/알림 진입 동선**
   - AppBar/공통 액션에서 라우팅 오류 없이 이동

5. **챌린지 기본 체험**
   - 피드 진입/스크롤
   - **로컬 mock 영상**으로 UI 몰입감·오버레이 겹침 검증
   - 영상 재생 실패 시 fallback으로 화면 붕괴 방지
   - 네트워크 스트리밍·버퍼링 검증은 **이번 스프린트 제외** (API 연동 단계)

### Shop

6. **Shop MVP 최소 동선**
   - 샵 인증
   - 공고 등록
   - 지원자 확인

### 품질·API

7. **품질 최소선**
   - 변경 범위 `dart analyze` 통과
   - 핵심 수동 QA 시나리오 체크
   - `flutter test`: **핵심 비즈니스 로직 Unit Test 필수**
     - 시간 계산, 스케줄 phase 판별, 금액/포인트 계산 등
     - UI/위젯 테스트는 이번 스프린트 **제외**

8. **API 전제 (mock vs real)**
   - **Real API 필수**: 인증, 공고, 스케줄/근무체크, 지원·지원자 확인
   - **Mock 허용**: 챌린지 피드 영상, 일부 부가 UI 데이터
   - Release 빌드에서 `useMockData` 강제 off 유지

---

## 3) Should (되면 포함)

1. 챌린지 텍스트 가독성/오버레이 미세 조정
2. 스케줄/근무체크 UI 세련화 추가 라운드
3. Shop 홈·서브 화면 완성도 개선 (Must 동선 외)
4. `ScheduleWorkSession` / WorkCheck ViewModel 단위 테스트 추가 커버리지

---

## 4) Won't (이번 스프린트 제외)

1. 고급 추천/랭킹 알고리즘
2. 대규모 실험 플랫폼(A/B infra)
3. 완전 자동화 백오피스
4. 수익화 실험 정식 런칭
5. **Shop 결제/정산 실연동**
6. **Shop 세부 알림 설정**
7. 챌린지 **네트워크 영상 스트리밍** 검증 및 CDN 연동

---

## 5) 이번 스프린트 Done 정의 (DoD)

- Must 항목 Spare + Shop 동선 수동 QA 통과 → **[MVP_QA_CHECKLIST.md](MVP_QA_CHECKLIST.md)** 에서 체크

- `dart analyze` 에러 0
- 핵심 Unit Test(스케줄 phase·근무체크 등) green
- 회귀 리스크/롤백 포인트 PR에 기록
- 관련 문서(`EXECUTION_ORDER`, 배포 체크리스트) 반영

---

## 6) Decisions (확정)

| 주제 | 결정 |
|------|------|
| Shop 범위 | 인증 · 공고 등록 · 지원자 확인만 Must. 결제·세부 알림은 Won't |
| 테스트 | UI/위젯 테스트 생략. 비즈니스 로직 Unit Test 필수 |
| 챌린지 영상 | 로컬 mock으로 UI·fallback 검증. 스트리밍은 백엔드 API 완성 후 |
