## 회원 역할 체계

`UserRole`은 `spare`(스페어) / `shop`(샵) / `admin`(관리자) 3종만 존재한다.
- **모델**은 별도 role이 아니라 스페어 계정의 서브타입(`SpareSubtype.model`)이다 — 회원가입 시 "모델"을 선택하면 `/model/*` 전용 화면 세트로 라우팅된다.
- **스토어**는 회원가입 화면에 선택지로 보이지만 실제로는 "준비중입니다" 안내만 뜨는 **미구현 placeholder**다. 스페어/샵 홈 메뉴의 "스토어" 항목도 마찬가지로 준비중 안내만 있다.

## 스페어(디자이너)

| 화면 | 기능 |
|---|---|
| 홈 / 공고 목록·상세 | 공고 탐색, 통합검색(공고·교육·공간·챌린지), 찜 |
| 스케줄 / 근무체크 | 확정된 근무 스케줄 확인, 근무 종료 후 근무체크로 보상 수령 |
| 채팅 | 샵과 1:1 채팅, **채팅 내 결제요청 카드**로 결제 진행 |
| 에너지 | 에너지(포인트) 구매·결제, 포인트 내역 |
| 챌린지 | 숏폼 영상 피드, 댓글, 크리에이터 프로필 |
| 교육 | 교육 목록/상세/신청, 에너지로 결제 |
| 공간대여 | 지역별 공간 목록·상세 |
| 계정 | 회원가입(전문가/모델), 본인인증, 아이디/비번 찾기, 프로필, 추천인, 리뷰, 알림 설정 |

## 모델 (스페어 서브타입, `/model/*`)

| 화면 | 기능 |
|---|---|
| 모델 홈 | 모델 전용 홈 |
| 매칭 | 조건 기반 스와이프 매칭, 날짜로 모델 찾기, 받은 관심/매칭 현황, 프로필 상세에서 수락·거절·채팅 |
| 모델 신청 | 신청 등록·현황 확인 |
| 프로필/스케줄 | 모델 프로필 관리, 스케줄 |

## 샵(미용실)

| 화면 | 기능 |
|---|---|
| 회원가입 | **사업자등록번호 OCR 인식 + 검증**, 대표/대리인 구분, 본인인증 |
| 인력 관리 | 지원자 목록·이력서 상세 |
| 공고 | 등록/목록/상세, **급구 노출**·**하이패스(오픈 예정) 노출** 유료 업셀 |
| 공간 | 내 공간 등록/수정, 예약 현황 |
| 등급/포인트 | VIP 등급 현황, 포인트 |
| 채팅/결제 | 스페어와 1:1 채팅, 구독·결제 내역 |
| 기타 | 교육 등록, 챌린지, 매칭 팁, 스케줄, 프로필/설정 |

## 관리자 (34개 화면, M1~M18 모듈)

| 영역 | 기능 |
|---|---|
| 회원 | 목록·상세, **정지/정지해제** |
| 공고/지원 | 목록·상세 |
| 매칭 | 모델-디자이너 매칭 목록·상세 |
| 결제/정산 | 결제 내역, 정산취소 요청 검토 |
| 신고/제재 | 신고 케이스(+채팅 로그 감사), 제재 실행·이력, 노쇼 신고 검토 |
| 콘텐츠 모더레이션 | 신고된 챌린지 영상·댓글 **숨김/삭제**. 별도로 회원 상세 화면에서는 챌린지 영상별 **승인(공개)/제한노출(노딱)/숨김/삭제** 4단계 검수 가능 |
| 인증심사 | 샵 사업자등록 OCR/국세청 증빙 검토 |
| 채팅 | 관리자-회원 1:1 고객센터 채팅 |
| 설정/기타 | 에너지, 포인트, 구독, 교육, 공간, 체크인, 비즈니스 설정, 알림 발송/템플릿, 레퍼런스 데이터, 감사 로그(읽기전용), 최근 활동, 대시보드 |

## 공통

- 포트폴리오(스페어·샵 공용 작업 포트폴리오)
- 알림 상세, 개인정보 처리방침
- **통합 로그인**: 스페어/샵 구분 없이 하나의 로그인 화면 사용

## 이번 세션에서 확인된 최근 기능 (구현 여부 코드로 확인)

| 기능 | 상태 | 근거 파일 |
|---|---|---|
| 채팅 내 결제요청 카드 | ✅ 구현 | `widgets/chat/payment_request_card.dart`, `payment_request_compose_sheet.dart` |
| 실시간 웹소켓 채팅 | ✅ 구현 | `services/chat_realtime_service.dart` (`/ws/chat`, JWT 인증, 재연결) |
| 연락처/전화번호 노출 차단 | ✅ 구현 | `services/contact_violation_service.dart` (3회 위반 시 대화방 삭제+샵 패널티) |
| 챌린지 영상 검수(승인/제한/숨김/삭제) | ✅ 구현 | `widgets/admin/admin_challenge_video_tile.dart` (회원 상세 화면에서) |
| 신고 콘텐츠 처리 | ⚠️ 부분 | `admin_content_screen.dart` — 숨김/삭제만, 승인/제한 액션 없음 |
| 샵 회원가입 사업자등록 인증 | ✅ 구현 | `signup_screen.dart` + OCR 모델/검증 유틸 |

## 도메인 모델 (`lib/models/`)

공고(job), 지원(application), 스케줄(schedule), 매칭(match_like/match_profile/model_designer_match), 챌린지(challenge_feed/comment/profile), 교육(education_enrollment/material), 포인트(point_transaction), 구독(subscription), 알림(notification), 공간대여(space_rental), 샵등급(shop_tier), 사업자등록 인증(business_registration_ocr_result 등), 유저(user).

채팅(Message)·결제(Payment)·에너지(EnergyTransaction) 등은 별도 모델 클래스가 아니라 `lib/services/`(chat_service.dart, payment_service.dart, energy_service.dart) 내부에 구현돼 있다.

- **스토어**: 회원가입·홈 전부 "준비중" 안내만 있음 (실기능 없음)
