# Figma AI 업무지시서: 인력별 페이지 디자인 디벨롭

## 1. 목표
- **대상**: 모바일 앱의 "인력별"(인력 목록) 페이지 전체 화면
- **요청**: 현재 구조와 큰 틀은 유지하되, **우리 앱 디자인 시스템에 맞춰** 시각적 퀄리티를 높여 달라.
- **제약**: 앱의 브랜드 색상(퍼플 계열), 간격·타이포 규칙을 준수할 것.

---

## 2. 앱 디자인 시스템 (준수 사항)

### 2.1 색상
- **Primary Purple**: `#9333EA` (메인 포인트, CTA, 선택 상태)
- **Primary Purple Light**: `#F3E8FF` (선택 배경, 연한 강조)
- **Purple 700**: `#7E22CE` (태그 텍스트 등)
- **배경**: 화이트 `#FFFFFF`, 리스트 배경 그레이 `#F9FAFB`
- **텍스트**: Primary `#111827`, Secondary `#6B7280`, Tertiary `#9CA3AF`
- **테두리**: `#E5E7EB`
- **그라데이션(아바타 등)**: Blue `#3B82F6` → Purple `#9333EA`

### 2.2 간격
- 8px 단위 권장: 8, 12, 16, 24, 32
- 카드/섹션 패딩: 12~16px 수준

### 2.3 모서리
- 버튼·칩·카드: 12px (radius-lg) 일관 사용
- 필·아이콘 버튼: 9999 (full round) 가능

### 2.4 타이포
- 페이지 제목: 굵게, 18~20px, primary purple 또는 dark gray
- 본문/라벨: 14px, medium/regular, gray 계열
- 작은 라벨/캡션: 12px 이하

---

## 3. 화면 구조 (Sequential – 이 순서로 레이아웃 유지)

아래 순서대로 블록이 쌓이는 구조를 유지하면서, 각 블록의 **스타일만** 디벨롭해 준다.

### Step 1: 상단 네비게이션 바 (본 페이지에서는 수정 제외)
- 백 버튼, 앱 로고 "HairSpare"(퍼플), 검색·메시지·알림 아이콘
- 흰 배경, 하단 1px 회색 구분선
- **지시**: 레이아웃 참고만 하고, 인력별 페이지에서는 상단바 영역을 비우거나 플레이스홀더로 두고, **그 아래 콘텐츠만 디자인**해도 됨.

### Step 2: 페이지 제목 영역
- **요소**: "인력별" 텍스트 1줄
- **현재**: 큰 제목, 보라색, 굵게
- **지시**: 앱 톤에 맞게 제목 스타일 정리 (자간, 크기, 색 조합). 필요 시 서브 타이틀용 작은 문구나 구분선 추가 검토.

### Step 3: 인원 수 + 새로고침 행
- **요소**: 왼쪽 "전체 인력 N명", 오른쪽 새로고침(원형 화살표) 아이콘
- **지시**: 톤 다운된 텍스트 스타일, 아이콘은 퍼플 포인트. 행 전체 정렬·여백을 8px 단위로 정리.

### Step 4: 필터 1행 – 드롭다운 버튼들
- **요소**: 가로 스크롤 가능한 행. 버튼 3종 (지역, 상세지역(선택 시만), 정렬)
- **라벨 예**: "지역", "상세지역", "정렬" / 옵션 예: "인기순", "최신순", "경력순", "완료건수순"
- **지시**:
  - 미선택: 흰 배경 + 연한 회색 테두리
  - 선택 시: 연보라 배경(#F3E8FF) + 퍼플 테두리/텍스트
  - 12px 라운드, 패딩 12~16px, 내부 화살표 아이콘
  - 버튼 간 간격 8px, 앱 전체 필터 톤과 통일

### Step 5: 필터 2행 – 카테고리 칩
- **요소**: 가로 스크롤 행. 칩 4개 – "전체"(👥), "스텝"(✂️), "디자이너"(💇), "면허인증"(✅)
- **지시**:
  - 선택: 연보라 배경 + 퍼플 테두리/텍스트, 약한 그림자 가능
  - 미선택: 연한 그레이 배경 + 회색 텍스트
  - 12px 라운드, 이모지+텍스트 정렬, 칩 간격 8px
  - 전체/스텝/디자이너/면허인증이 한 줄에 보이도록 비율·패딩 조정

### Step 6: 인력 카드 리스트 (핵심 디벨롭)
- **카드 1장 구조 (위→아래)**:
  1. **좌측**: 원형 프로필 이미지 (48px 또는 60px). 이미지 없을 때 그라데이션(블루→퍼플) + 이름 첫 글자
  2. **우측 상단 행**: 이름(굵게) + "인기" 배지(노란·오렌지 계열, 별 아이콘) + "면허인증" 배지(연보라 배경, 보라 텍스트) – **인기와 면허인증 사이에 여백 유지**
  3. **그 아래**: "경력 N년 · 완료 N건" (회색 작은 글씨)
  4. **그 아래**: 전문 태그 칩 2~3개 (예: 컷, 펌, 셰이빙) – 연보라 배경, 보라 텍스트, 작은 라운드
  5. **맨 아래**: "따봉 N개"(퍼플 강조) · "리뷰 N개" · "본인인증"(선택)
- **지시**:
  - 카드 전체: 흰 배경, 12px 라운드, 1px 연한 테두리, 약한 그림자
  - 카드 간 세로 간격 8~12px
  - 프로필 원형과 텍스트/배지 간격 8~12px
  - "인기" 배지는 **프로필 사진과 겹치지 않게** 이름 옆(오른쪽)에만 배치
  - 타이포 위계(이름 > 경력 > 태그 > 따봉·리뷰) 명확히, 색은 앱 팔레트만 사용

### Step 7: 빈 상태·에러 상태 (선택)
- **빈 상태**: "조건에 맞는 인력이 없습니다" + 일러스트 또는 아이콘
- **에러**: "오류가 발생했습니다" + 재시도 버튼
- **지시**: 배경 그레이, 중앙 정렬, 퍼플 포인트 1개(버튼 또는 아이콘)만 넣어도 됨.

---

## 4. 디벨롭 시 유의사항
- **일관성**: 다른 샵 탭(공고 목록, 스케줄 등)과 필터·카드 스타일이 어울리도록 퍼플+그레이 계열로 통일.
- **터치 영역**: 버튼·칩·카드 최소 높이 44px 근처 유지.
- **스크롤**: 필터 1행·2행은 가로 스크롤 가능하다고 가정하고, 오른쪽 여백 16px 정도 확보.
- **다크 모드**: 지시서에서는 라이트 모드만 정의. 필요 시 나중에 다크 팔레트 별도 지시.

---

## 5. 산출물 요청
- **인력별 페이지** 메인 스크린 1종 (상단바 제외 또는 플레이스홀더 포함)
- **재사용 컴포넌트**: 필터 드롭다운 버튼(기본/선택), 카테고리 칩(기본/선택), 인력 카드 1종(컴팩트 버전 기준)
- **상태**: 필터 선택/미선택, 카드 기본 상태만 있어도 됨 (호버·프레스는 선택)
- **스펙**: 색상 HEX, 간격(px), radius(px)는 위 2번 시스템 숫자에 맞춰 주석 또는 스타일 이름으로 명시

## 6. 참고 문구 (Figma AI에 복사용)
```
인력별 페이지를 다음 조건으로 디자인해 줘.

1) 디자인 시스템: Primary Purple #9333EA, Purple Light #F3E8FF, 배경 화이트/그레이 #F9FAFB, 텍스트 #111827/#6B7280. 간격 8px 단위, radius 12px.

2) 화면 순서: 페이지 제목 "인력별" → "전체 인력 N명" + 새로고침 → 가로 스크롤 필터 드롭다운(지역, 정렬) → 가로 스크롤 칩(전체, 스텝, 디자이너, 면허인증) → 인력 카드 리스트.

3) 인력 카드: 좌측 원형 프로필(블루-퍼플 그라데이션), 우측 이름 + 인기 배지 + 면허인증 배지(간격 있게) + 경력·완료 건수 + 전문 태그(컷, 펌 등) + 따봉·리뷰·본인인증. 카드 흰 배경, 12px 라운드, 연한 테두리·그림자.

4) 필터/칩 선택 시 연보라 배경+퍼플 테두리, 미선택 시 회색 톤. 인기 배지는 프로필과 겹치지 않고 이름 옆에만 배치.

위 구조와 색상·간격을 지키면서 보기 좋게 다듬어 줘.
```
