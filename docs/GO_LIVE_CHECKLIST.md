# HairSpare Go-Live Checklist v2 (Team-Ready)

> 출시 직전/당일/직후에 팀이 그대로 따라갈 수 있는 운영 체크리스트.

---

## 1) 출시 전 (T-7 ~ T-1)

## 1.1 범위/품질
- [ ] 이번 릴리즈 범위 확정 (Must only)
- [ ] Known Issues 정리
- [ ] Sev1/Sev2 오픈 이슈 0건 또는 승인된 예외

## 1.2 기술 검증
- [ ] `dart analyze` 통과
- [ ] 테스트 통과(도입 범위)
- [ ] 수동 QA 완료
  - [ ] 로그인/세션 복원
  - [ ] 공고 목록/상세/지원
  - [ ] 스케줄/근무체크(종료시간 기준)
  - [ ] 채팅/알림 라우팅
  - [ ] 챌린지 영상 재생/스크롤

## 1.3 배포 준비
- [ ] 버전/빌드번호 증가
- [ ] 프로덕션 `--dart-define` 검증
- [ ] `USE_MOCK_DATA=false` 확인
- [ ] 스토어 메타데이터 최신화(설명/스크린샷/정책 링크)
- [ ] 릴리즈 노트 작성

---

## 2) 출시 당일 (T0)

## 2.1 배포
- [ ] 스토어 제출/릴리즈 트리거
- [ ] 릴리즈 태그 기록 (`vX.Y.Z`)
- [ ] 배포 시간/담당자 기록

## 2.2 즉시 스모크 테스트 (10~15분)
- [ ] iOS 실기기 1대 이상
- [ ] Android 실기기 1대 이상
- [ ] 핵심 플로우 5개 재검증

---

## 3) 출시 후 24시간 (T+1)

## 3.1 모니터링
- [ ] 크래시율
- [ ] API 실패율
- [ ] 로그인 실패율
- [ ] 핵심 퍼널(진입→지원/체크) 이상치

## 3.2 대응
- [ ] 이슈 triage (Sev1/2/3)
- [ ] 필요 시 `hotfix/*` 즉시 생성
- [ ] 사용자 공지 필요 여부 판단

---

## 4) 롤백 트리거

- [ ] 로그인 불가/핵심 기능 마비 (Sev1)
- [ ] 특정 OS/버전 크래시 급증
- [ ] 결제/정산 치명 오류

롤백 절차:
1. 이전 안정 태그로 재배포
2. 릴리즈 일시 중단
3. hotfix 계획 및 커뮤니케이션

---

## 5) 출시 회고 (T+3 ~ T+7)

- [ ] 목표 KPI 달성 여부 리뷰
- [ ] 사용자 피드백 정리
- [ ] 다음 릴리즈 Top3 확정
- [ ] 체크리스트 개선점 반영

---

## 6) 담당 역할 매핑 (권장)

- Release Owner: 배포 진행/의사결정
- QA Owner: 체크리스트 실행/결과 보고
- Dev Owner: hotfix 대응/원인 분석
- PM Owner: 범위/KPI/커뮤니케이션
